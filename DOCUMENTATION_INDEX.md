## Documentation Maintenance

### For Package Maintainers

When updating the package:
1. Update relevant documentation files
2. Add entry to [CHANGELOG.md](CHANGELOG.md)
3. Review examples for correctness
4. Check for broken links
5. Update version references

### Documentation Checklist

**For new features:**
- [ ] Add to [API_REFERENCE.md](API_REFERENCE.md)
- [ ] Add example to [USAGE_GUIDE.md](USAGE_GUIDE.md)
- [ ] Update [README.md](README.md) if major feature
- [ ] Add to [CHANGELOG.md](CHANGELOG.md)

**For bug fixes:**
- [ ] Update [TROUBLESHOOTING.md](TROUBLESHOOTING.md) if relevant
- [ ] Add to [CHANGELOG.md](CHANGELOG.md)
- [ ] Check examples still work

**For API changes:**
- [ ] Update [API_REFERENCE.md](API_REFERENCE.md)
- [ ] Update affected examples
- [ ] Add migration guide if breaking change
- [ ] Update [CHANGELOG.md](CHANGELOG.md)
